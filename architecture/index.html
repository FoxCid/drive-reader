    Architecture

    Vue d’ensemble

    Le pipeline fonctionne en deux phases complémentaires :

    1) Phase 1 — Contexte (role-aware)
    - retrieve_context_chunks(role)
    - summarize_context_for_role()
    - Si le contexte est insuffisant → enrich_with_web() (hook)

    2) Phase 2 — Général (rappel large + scoring)
    - retrieve_general_chunks()
    - Normalisation : to_standard_chunk() + enhance_chunk()
    - Scoring & tri : score_sort_and_rerank()
    - Sélection : select_final_chunks() (diversification soft, Top-1 préservé)
    - Synthèse : synthesize_answer() / generate_final_answer()

    Résultat : Réponse + Sources + Debug.

    Modules principaux

    • two_step/api.py : orchestration semantic_search_two_step() et run_two_step().
    • two_step/scoring.py : compute_total_scores(), score_sort_and_rerank(), select_final_chunks().
    • two_step/rerank_utils.py : rerank_head() + fallback lexical si le cross-encoder est indisponible.
    • Builders :
    • build_index_incremental_safe_fixed.py (index “docs” généraux)
    • build_context_index_incremental.py (index “context” par rôle)
    • Config : faiss_config.py (+ variables d’environnement).

    Deux étapes détaillées

    Phase 1 — Contexte

    • Récupération de snippets dédiés au rôle et résumé synthétique.
    • Si insuffisant ➜ enrich_with_web() pour injecter des mots-clés/signaux externes.

    Phase 2 — Général

    • Gros recall (FAISS/BM25), normalisation uniforme des chunks.
    • Scoring = FAISS/BM25 + bonus entités + fraîcheur + feedback.
    • Rerank (cross-encoder si dispo, sinon fallback lexical).
    • Diversification par source et préservation du Top-1.
