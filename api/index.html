    API

    Points d’entrée

    two_step.api.semantic_search_two_step(...)

    Wrapper tolérant qui accepte query ou question.

    Signature (effective)
    semantic_search_two_step(query: str, role: str = "", top_k: int = 8, model: Optional[str] = None, retrieval_only: bool = False, **kwargs) -> Union[Tuple[str, List[str], Dict, Optional[str]], List[Dict], Dict]

    Comportement - retrieval_only=True → renvoie la liste de chunks (pour tests golden). - Sinon → renvoie (answer, sources, used_debug, summary_context).

    Kwargs pris en charge - mode: 1 (phase 1 preview), 1.5 (preview + résumé), 2 (pipeline complet). - Pass-through vers l’impl interne si présent.

    Sorties (mode complet) - answer: str - sources: List[str] (chemins normalisés) - used_debug: Dict (ex: {"chunks_used": [...], "entities": [...], ...}) - summary_context: Optional[str]

    two_step.api.run_two_step(...)

    Pipeline orchestration (toujours tuple en sortie).

    Étapes 1. extract_entities_keywords() 2. Contexte : retrieve_context_chunks() → summarize_context_for_role() - si context_is_sufficient(summary) est False → enrich_with_web() 3. Général : retrieve_general_chunks() 4. Normalisation : to_standard_chunk() + enhance_chunk() 5. Scoring : score_sort_and_rerank() 6. Sélection : select_final_chunks() (diversification soft, Top-1 préservé) 7. Synthèse : synthesize_answer() ou generate_final_answer()

    Hooks & extensions

    • context_is_sufficient(summary: str) -> bool
      (par défaut: True si non défini)
    • enrich_with_web(query, role) -> List[str]
      : ajoute des mots-clés externes quand le contexte est faible.
    • rerank_utils.rerank_head(chunks, top_k)
      : rerank “head” (utilise CE si dispo, sinon fallback lexical).

    Schéma “chunk” normalisé

    ```json { "text": "…", "source": "data/docs/…", "metadata": { "faiss_score": 0.0, "bm25_score": 0.0, "total_score": 0.0, "filename": "…", "freshness_days": 12, "entity_bonus": 0.1, "feedback_penalty": 0.0 } }

    Note: la phase2_query est désormais dédupliquée et bornée (~96 tokens) pour éviter les requêtes trop verbeuses.
